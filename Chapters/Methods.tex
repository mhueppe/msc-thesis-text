\chapter{Methods}\label{methods}
This chapter presents the methodological framework used to model the relationship between proteins and crystallization conditions. 

First, \autoref{sec:methods/feature_engineering} presents the set of feature representations used to represent the protein and condition. 
Building on these representations, a two-tower model architecture is introduced in \autoref{sec:methods/model_architecture} to jointly embed proteins and conditions into a shared latent space, enabling similarity-based ranking of candidate conditions. The contrastive learning training procedure and optimization of hyperparameters is described in \autoref{sec:methods/training}. To properly evaluate this model the \autoref{sec:methods/evaluation} further details the data splitting strategy and evaluation metrics.

Together, these components define an interpretable, scalable and flexible pipeline for learning relationships between protein features and crystallization outcomes.
